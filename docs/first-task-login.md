# Перший таск: Вхід на сайт

## Чому це найпростіший старт

`Вхід на сайт` найзручніше брати першим, бо він:

- не залежить від кошика;
- не залежить від авторизації через Google;
- не вимагає повного профілю користувача на першому етапі;
- дозволяє швидко показати робочий результат команді.

## Мінімальний обсяг першої реалізації

Для першої версії достатньо:

1. Екран з логотипом або назвою `Rozetka`.
2. Поле `Email`.
3. Поле `Пароль`.
4. Кнопка `Увійти`.
5. Базова валідація:
   - email має бути коректного формату;
   - пароль не може бути порожнім;
   - бажано перевіряти мінімальну довжину.
6. Повідомлення про помилку під полями або над кнопкою.
7. Чернетковий перехід на наступний екран після успішного входу.

## Що можна відкласти на пізніше

Це не обов'язково для першого кроку:

- справжній бекенд;
- JWT або інші токени;
- Google-вхід;
- відновлення пароля;
- запам'ятовування користувача між сесіями.

## Як можна поділити роботу між трьома людьми

- Учасник 1: дизайн і верстка екрана входу.
- Учасник 2: логіка валідації форми.
- Учасник 3: навігація і підготовка базової структури проєкту.

## Результат першого етапу

Після завершення цього таску команда вже матиме:

- окремий стартовий екран;
- першу завершену user-flow;
- основу для реєстрації та відновлення пароля.
